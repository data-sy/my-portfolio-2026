description: Create a commit from currently staged changes only (do not stage anything new).
---

Analyze only the currently staged changes — do NOT run `git add`.

**1. Pre-check:**
- Run `git diff --cached --stat`
- If nothing is staged, inform the user and exit: "No staged changes found. Use `git add` first."

**2. Analysis:**
- Review staged changes: `git diff --cached --stat` and `git diff --cached`
- Understand what was added, modified, or deleted

**3. Commit Message Rules:**
- Title: Conventional Commits (feat, fix, refactor, perf, docs, test, chore)
- Title: English, lowercase after prefix, no period at end
- Body: English, bullet points with `-`, each line starts with an uppercase letter
- One blank line between title and body
- Write the message to a temp file to preserve multi-line formatting
- NEVER include `Co-Authored-By`, `Generated by`, or any AI attribution

**4. Format:**
```
type: concise description in english

- Detailed change description 1
- Detailed change description 2
```

**5. Execution:**
```bash
cat << 'EOF' > /tmp/commit_msg.txt
{generated message}
EOF
git commit -F /tmp/commit_msg.txt
rm /tmp/commit_msg.txt
```

**6. After commit:**
- Show the result with `git log --oneline -1`
