# 프로젝트 4: Skeleton Gym

## 기본 정보
- 프로젝트 이름: Skeleton-Gym
- 영문 이름: skeleton-gym
- 버전: v1.0.0
- 기간: 2021.10 ~ 11 (4주)
- 역할: BE 40%, 발표 담당 (5인 개발: BE 3명, FE 2명)
- 기술 스택: Python 3.8, Flask, OpenCV, MediaPipe

## 프로젝트 개요
- 서비스 설명: 영상에서 관절점을 추출해 운동 자세와 횟수를 분석하는 프로그램. 실시간으로 자세를 교정하고 트레이닝 기록을 관리하여 안전한 운동을 도움
- 개발 배경: 헬스장에서 잘못된 자세로 운동하다 부상당하는 경우가 많음. 트레이너 없이도 자세를 교정받을 수 있는 AI 기반 서비스 개발
- 주요 기능: 실시간 영상에서 인간 골격 추출, 운동 횟수 자동 측정, 자세 교정 피드백

## 정량적 성과
- 성과 1: 공통 모듈 도입으로 운동별 로직 구현 시간 83.33% 단축 (3일 → 0.5일)
- 성과 2: 핵심 비즈니스 로직 75% 개발 (운동 횟수 측정 + 자세 교정 로직 3개 중 2개)
- 성과 3: 최종 발표회 최우수상(2등) 수상

## 인사이트
- 배운 점 1: 작은 코드 변화(공통 모듈)로 개발 속도를 크게 향상시킬 수 있음
- 배운 점 2: 문제 해결은 거창한 것이 아니라, 불편함을 느낀 부분에서 문제를 정의하고 해결 방안을 고민하는 과정임

## 트러블슈팅 목록

### 트러블슈팅 1 (Type: A)
- 제목: 공통 모듈 도입으로 개발 속도 83.33% 단축
- 문제 상황: MediaPipe Pose가 관절 위치를 직교좌표계(x, y, z)로 반환하여, 운동별로 매번 새로운 로직을 작성해야 하는 비효율 발생
- 해결 과정:
  - 시도 1: 인간 움직임이 관절 중심 회전이라는 점 고려, 구면좌표계(r, θ, ɸ)로 변환 → 로직 구현이 수월해졌으나 여전히 운동마다 별도 구현 필요
  - 시도 2: 세 개의 관절 위치를 파라미터로 받아 각도를 반환하는 공통 모듈 개발
- 결과: 운동별 로직 구현 시간 3일 → 0.5일 (83.33% 단축)
- 배운 점: 반복되는 패턴을 추상화하면 개발 생산성이 크게 향상됨. 좌표계 변환이라는 수학적 접근이 코드 설계에 도움이 됨
