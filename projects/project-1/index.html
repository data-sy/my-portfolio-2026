프로젝트 개요

대규모 E-Commerce 플랫폼을 가정하여, 회원 100만 명 · 일평균 주문 5만 건 규모의 트래픽 환경에서 발생하는 성능 병목을 재현 · 측정하고 개선한 프로젝트입니다.

실무에서 마주칠 대규모 트래픽 상황을 미리 경험하고, 성능 병목을 직접 재현 · 측정 · 개선하는 과정을 통해 백엔드 성능 튜닝 역량을 키우기 위해 시작했습니다.

기술 스택

Java Spring Boot JPA Redis MySQL (RDS) Terraform AWS ECR

기간

2026.01 (3주)

역할

1인 개발 (설계, 구현, 테스트, 인프라 구축 전체)

주요 기능

  • 상품 목록 조회 (카테고리 필터 + 정렬 + 페이징)
  • 주문 상세 조회 (연관 엔티티 포함)
  • 실시간 판매량 랭킹 TOP 100
  • 선착순 쿠폰 발급 / 재고 차감 (동시성 제어)

정량적 성과

  • 상품 목록 조회 93% 개선: DB 인덱스 최적화를 통해 2.5초 → 180ms
  • 주문 상세 조회 81% 개선: N+1 문제 해결로 800ms → 150ms
  • 실시간 랭킹 98% 개선: Redis Sorted Set으로 200ms → 5ms
  • 동시성 제어 전략 확보: 시나리오별 락 전략 선택 기준 수립

인사이트

  • 인덱스 설계: 인덱스는 "컬럼 추가"가 아니라 WHERE 조건 + 정렬/페이징 패턴까지 포함한 설계 문제이며, EXPLAIN 기반 검증을 통해 각 시도의 효과를 정확히 측정해야 함
  • ORM 사용 시 주의: 편의성만 보지 말고 실제 실행 쿼리 수와 I/O 비용을 반드시 확인해야 함
  • 자료구조 선택: 랭킹처럼 읽기 빈도가 높고 정렬이 핵심인 문제는 RDB 집계보다 적합한 자료구조를 쓰는 게 효과적
  • 동시성 제어: "정답 락"이 아니라 정확성 · 성능 · 사용자 경험 · 인프라 환경을 함께 보고 선택해야 함

기술 문서

DB 인덱스 최적화를 통한 상품 목록 조회 성능 93% 개선 N+1 문제 해결로 주문 상세 조회 성능 81% 개선 Redis Sorted Set으로 실시간 랭킹 조회 98% 개선 동시성 제어 비교로 상황별 락 전략 선택 경험
