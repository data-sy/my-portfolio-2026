## 트레이드오프 맥락 (면접 대비용)

상황에서 핵심 제약 조건:
- 제약 1: 동시 1000명 요청 → 높은 동시성 환경
- 제약 2: 정확히 100명만 발급 → 정합성 필수
- 제약 3: 다중 서버 배포 환경 → JVM 락 불가

각 선택지를 고려한 이유:
- synchronized: 가장 단순하지만 분산 환경 미지원으로 탈락
- DB 비관적 락: 분산 환경 지원하지만 커넥션 점유 문제
- Redis Lua Script: DB 부하 분리, 원자적 처리 가능

최종 선택의 결정적 근거:
- DB 비관적 락 대비: 커넥션 풀 고갈 위험 없음
- 이미 Redis 인프라 존재: 추가 도입 비용 최소
- Lua Script: 별도 분산락 없이도 원자적 수량 관리 가능

만약 상황이 달랐다면:
- Redis 미사용 환경 → DB 비관적 락 + 커넥션 풀 튜닝
- 동시 요청이 10명 이하 → synchronized로 충분
- 정합성보다 처리량 우선 → 낙관적 락 + 재시도
