# Troubleshooting A: 문제해결형

## 구면 기하학 적용으로 거리 계산 오차율 88% 개선
| key | value |
|---|---|
| title | 구면 기하학 적용으로 거리 계산 오차율 88% 개선 |
| problem | 반경 조회 시 평면 유클리드 거리 공식 사용으로 오차율 32% 발생. 실제 반경 밖 장소가 결과에 포함되거나 반경 내 장소가 누락 |
| try_1_title | 유클리드 거리 공식 (기존) |
| try_1_desc | 위도·경도를 평면 좌표로 취급하여 √((x₂-x₁)² + (y₂-y₁)²) 계산 |
| try_1_result | 오차율 32% |
| try_1_limit | 지구 곡률 무시. 위도 1도와 경도 1도의 실제 거리가 위치에 따라 다름을 반영하지 못함 |
| try_2_title | 위도·경도 보정 계수 적용 |
| try_2_desc | 위도 1도 ≈ 111km, 경도 1도 ≈ 111km × cos(위도) 보정 후 유클리드 계산 |
| try_2_result | 오차율 약 15% |
| try_2_limit | 근거리에서는 개선되나 거리가 멀어지면 평면 근사의 한계 여전 |
| try_3_title | Haversine 공식 (구면 기하학) |
| try_3_desc | 지구를 구로 가정, 두 지점의 위도·경도로 대원 거리(great-circle distance) 계산 |
| try_3_result | 오차율 3.69% (88%↓) ✓ |
| try_3_limit | 지구 타원체 미반영이나 위치 서비스에는 충분한 정확도 |
| final_result | 오차율 32% → 3.69% (88%↓) |
| insight_1 | 위도·경도는 평면 좌표가 아닌 구면 좌표. "좌표니까 유클리드 거리"라는 직관이 틀렸으며, 데이터의 물리적 의미를 이해해야 올바른 공식을 선택할 수 있음 |
| insight_2 | 문제를 인식한 뒤 "왜 오차가 생기는가"를 수학적으로 분석한 것이 해결의 시작점. 직관적 해결(보정 계수)보다 원리 기반 해결(Haversine)이 근본적이고 범용적 |
| followup_q1 | Haversine 공식의 수학적 원리를 설명할 수 있나? |
| followup_q2 | 오차율 측정을 위한 기준 데이터는 어떻게 구했나? |

상황: 현재 위치 기반 반경 N km 내 플로깅 장소 조회
SELECT * FROM places
WHERE 거리(현재위치, 장소위치) <= N;

- 왜? 좌표가 (위도, 경도)로 표현되니 직관적으로 피타고라스 정리 적용
- 한계: 위도 1도 ≈ 111km이지만 경도 1도는 위도에 따라 가변 (서울 약 89km, 적도 111km)

시도 2: 보정 계수
- 왜? 경도의 가변성을 cos(위도)로 보정하면?
- 한계: 근거리 개선되나 평면 근사의 근본 한계 (지구는 구)

시도 3: Haversine 공식
- 왜? 지구를 구로 가정하고 대원 거리 계산
- 핵심: 두 점 사이의 "곡면 위 최단 거리"를 직접 계산
