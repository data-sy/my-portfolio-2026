  - MMT에서는 EXPLAIN 없이 직관으로 시행착오

### 생략한 섹션:
- SECTION 2-2: 인덱스 적용은 상황별 선택지가 명확하지 않아 생략

## 예시 2: 04-concurrency (쿠폰 동시성)

### 사용한 섹션:
- SECTION 1: 전체 작성 (synchronized → DB 락 → Redis 락)
- SECTION 2-2: **각 시도의 적합한 상황** 상세 작성
  - synchronized: 단일 서버, 소규모
  - DB 락: 분산 환경 + 트랜잭션 보장
  - Redis 락: 분산 환경 + 고빈도 요청
- SECTION 2-3: **테스트 검증 방법** 상세 작성
  - 로컬 ExecutorService
  - Docker Compose 3개 인스턴스
  - JMeter 1000명 부하 테스트
- SECTION 3: 전체 작성

### 생략한 섹션:
- SECTION 2-1: 코드가 간단해서 생략
- SECTION 4: 프로젝트 간 연결 없음
