## 비교 분석 맥락 (면접 대비용)

왜 이 비교를 했나:
- Spring 백엔드 → SwiftUI iOS 앱으로 개발 영역 확장
- "같은 기능인데 왜 고민 포인트가 다르지?"라는 질문에서 시작
- 서버 개발자가 클라이언트를 경험한 관점 변화 정리

비교 관점 선정 이유:
- 상태 관리: 가장 근본적인 차이 (Stateless vs Stateful)
- 데이터 저장: 서버 DB vs 로컬 저장소
- 에러 처리: 내부 처리 vs 사용자 직접 노출
- 배포/업데이트: 즉시 반영 vs 심사 + 사용자 의존

결론 도출 과정:
- 백엔드: 보이지 않는 곳에서 데이터와 성능에 집중
- 모바일: 사용자 눈앞에서 경험과 반응성에 집중
- 두 관점을 이해하면 더 나은 협업과 설계 가능

인사이트 연결:
백엔드만 해봤을 때는 "API 응답만 잘 주면 된다"고 생각했지만,
클라이언트를 해보니 "이 응답을 어떻게 화면에 보여줄까"를 고민하게 됨.
이 관점 전환이 API 설계의 품질을 높여줌.
