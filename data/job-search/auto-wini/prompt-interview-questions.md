# 면접 예상 질문 생성 프롬프트

> Claude Code에서 사용. 이 프롬프트와 함께 트러블슈팅 md 파일들이 있는 폴더 경로를 지정할 것.
> ⚠️ 이 프롬프트는 **질문 리스트만** 생성한다. 모범답안은 별도 요청으로 진행할 것.

## 면접 전제 조건
- 면접관은 지원자의 이력서와 포트폴리오 PDF를 사전에 읽은 상태에서 질문한다
- 면접은 "간단히 자기소개 부탁드립니다"로 시작될 가능성이 높다. 자기소개 요청도 질문에 포함할 것
- 자기소개 이후에는 이력서 내용을 전제한 구체적 질문이 이어질 가능성이 높다
- 질문을 생성할 때 이 전제를 반영할 것

## 지원자 배경 (면접관이 이력서/포트폴리오에서 파악 가능한 정보만)
- 전직 수학교사, 36세, 백엔드 개발자로 커리어 전환
- 실무 경험 없음 (부트캠프 수료 후 취업 준비 중)
- 주력 스택: Java / Spring Boot
- 포트폴리오 프로젝트: Performance Tuning(DB 최적화), MMT(수학 학습 플랫폼, DKT AI + Neo4j + Spring Boot), QuickLabelTimer(iOS 앱, App Store 출시), Plogging(JSP/Servlet 커뮤니티), FormCheck
- 면접관은 교직 경력이 정확히 몇 년인지, 임용시험 준비 기간이 얼마인지는 모른다. 나이(36)와 "전직 수학교사"라는 사실만 알고 있으므로, 공백기나 경력 기간에 대해 질문할 가능성이 있다.
- 강점 키워드는 별도로 제공하지 않는다. 면접관이 포트폴리오 md 파일들을 읽고 스스로 판단하도록 할 것. 질문도 md 파일 내용을 근거로 생성할 것.

## 지원 회사: 오토위니 (Autowini)

### 회사 개요
- 2014년 설립, 중소기업 (사원수 약 50명, 자회사 포함 ~80명)
- 매출 약 150~174억 (2024~2025)
- 서울 강남구 도곡동 소재
- 한국 중고차 수출 B2B 플랫폼 분야 1위
- 100개국 이상, 10만 대 이상 차량 판매 실적
- 평균 연령 30대 초반, 능력 우선주의 표방
- 별도 기술 블로그 없음

### 서비스 라인업
- autowini.com: 메인 중고차 수출 플랫폼 (셀러-바이어 직접 연결, 글로벌 B2B)
- 옥션위니: 글로벌 폐차/중고차 실시간 경매 플랫폼 (2019 런칭, 2024 렌터카 옥션 추가)
- 파츠위니: 자동차 부품 플랫폼 (2018 런칭)
- 위니로지스: 물류 전문 자회사 (2022 설립, 선적/운송 서비스)

### 기술 스택 (JD 기반)
- Backend: Java/Spring
- DB: Oracle, MySQL, Redis
- Infra: AWS
- 도구: CodeRabbit(AI 코드리뷰), DevOps 문화 정착 중
- 개발팀장 스택 참고: jQuery, JavaScript, MySQL, Oracle, Spring, Java

### 회사 수준 및 면접 난이도 판단
- 기업 규모: 중소기업 (50인 규모 자체서비스 회사)
- 채용 포지션: 신입/주니어 백엔드 개발자
- 면접 단계: 1차 (코딩테스트 Java 1문제 + JS 2문제 + 대면 기술면접)
- 예상 면접 수준: 대기업 면접처럼 시스템 설계나 아키텍처 심화를 물어보기보다는, "기본기를 이해하고 있는지" + "포트폴리오에서 왜 그런 선택을 했는지" + "함께 일할 수 있는 사람인지"를 확인하는 수준
- 기술 블로그가 없고 개발팀 규모가 작은 점을 감안하면, 최신 트렌드보다 실용적 기본기 위주 질문이 예상됨

## 페르소나 (3명이 각각 질문 생성)

### 페르소나 1: 시니어 백엔드 개발자 (경력 7년+)
- 역할: 기술 깊이 검증
- 관심사: 설계 판단 근거, 트레이드오프 이해도, 실제 동작 원리
- 질문 스타일: 포트폴리오에서 출발해서 점점 깊이 파고드는 꼬리질문
- 전제: 지원자의 포트폴리오 PDF와 GitHub을 사전에 확인한 상태

### 페르소나 2: 팀 리드
- 역할: 협업 가능성, 학습 속도, 성장 잠재력 평가
- 관심사: 문제 해결 과정, 팀 경험, 피드백 수용, 모르는 걸 어떻게 해결하는지
- 질문 스타일: 상황 기반 ("~한 상황에서 어떻게 하겠어요?")
- 전제: 이력서의 수학교사 경력과 팀 프로젝트 리드 경험을 인지한 상태

### 페르소나 3: 인사 담당자
- 역할: 컬처핏, 동기, 지속 가능성 평가
- 관심사: 전환 동기, 36세 나이와 경력 공백에 대한 궁금증, 늦은 시작에 대한 태도, 회사 선택 이유
- 질문 스타일: 서사/태도 중심, 압박 질문 포함
- 전제: 이력서에서 "전직 수학교사"와 나이(36)만 확인한 상태. 교직 경력이 정확히 몇 년인지, 그 사이에 뭘 했는지는 모름

## 질문 카테고리

### A. 프로젝트 기반 질문
- 각 md 파일의 context, try 과정, insight, 주석 내 면접 대비 메모를 근거로 생성
- 이미 followup_q로 존재하는 질문은 그대로 포함하되 (기존) 태그 표기
- 꼬리질문 체인도 생성 (Q → 예상 꼬리 Q1 → 꼬리 Q2)

### B. CS 기본 질문
아래 주제를 반드시 포함:
- 네트워크: HTTP/HTTPS 동작 원리(TLS 핸드셰이크, 대칭키/비대칭키), TCP/UDP, REST API 설계 원칙, HTTP 상태코드, 쿠키/세션 차이
- DB: 트랜잭션 ACID, 인덱스 B+Tree 동작 원리, 정규화/반정규화, 락(낙관적/비관적), N+1 문제
- Java: JVM 메모리 구조, GC 기본, 컬렉션 프레임워크(ArrayList vs LinkedList, HashMap 동작 원리), 인터페이스 vs 추상클래스, 제네릭
- Spring: Bean 라이프사이클, IoC/DI, AOP, Filter vs Interceptor vs AOP 차이, Spring MVC 요청 처리 흐름
- 기타: Git 브랜치 전략, Docker 기본 개념

### C. 서사/컬처핏 질문
- 자기소개 (1분 자기소개 요청, 또는 "간단히 본인 소개 부탁드립니다")
- 커리어 전환 동기 (수학교사 → 개발자)
- 나이(36)와 경력 공백에 대한 질문 ("교사를 그만두고 개발까지 오기까지 어떤 과정이 있었나요?" — 면접관은 구체적 사정을 모르는 상태에서 물어봄)
- 실무 경험 부재에 대한 대응
- 팀 협업 경험
- 회사 선택 이유 ("왜 오토위니인가?")
- 압박 질문 (늦은 시작, 체력/지속성, 실패 경험, "교사를 왜 그만뒀나요?", "다시 교사로 돌아갈 생각은 없나요?")

## 중요도 기준
- ☆ 상: 출제 확률 80%+ 또는 답변 못하면 탈락 리스크. **이것부터 준비할 것.**
- ○ 중: 출제 확률 50%+ 또는 잘 답하면 차별화 가능. 시간 되면 준비.
- △ 하: 출제 가능성 낮지만 준비하면 플러스. 여유 있을 때만.

## 기타 규칙
- 프로젝트 기반 질문은 md 파일 수 × 최소 5개 이상
- CS 기본 질문은 카테고리당 최소 3개
- 서사 질문은 최소 10개 (자기소개 포함)
- 질문 총 개수는 50~80개 범위
- 중요도 ☆ 상이 전체의 30~40% 되도록 균형 맞추기
- "오토위니 연결 포인트" 컬럼: 해당 질문이 오토위니의 어떤 서비스/기술과 연결되는지 한 줄로 표기 (연결 안 되면 비워둠)
