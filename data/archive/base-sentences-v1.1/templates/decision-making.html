의사결정 과정
synchronized
├─ 다중 서버 환경 → ❌ JVM 단위 락으로 정합성 보장 불가
└─ 제외
DB 비관적 락
├─ 분산 환경 지원 → ✅ 정합성 보장 가능

└─ 보류
Redis 분산락
├─ 분산 환경 지원 → ✅ 정합성 보장
├─ 동시 1000+ 요청 → ✅ 높은 처리량
├─ Redis 이미 보유 → ✅ 추가 인프라 비용 0
└─ ✓ 선택

인사이트
동시성 문제는 "어디서 락을 잡을 것인가"가 핵심. JVM → DB → Redis 순으로 락의 범위가 확장되며, 분산 환경에서는 애플리케이션 외부에 락을 두어야 함
