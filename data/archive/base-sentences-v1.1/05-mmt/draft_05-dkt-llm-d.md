# Troubleshooting D: 트레이드오프형

## DKT 수치 출력 한계 — LLM 자연어 피드백 전환 검토 (v2)
| key | value |
|---|---|
| title | DKT 수치 출력 한계 — LLM 자연어 피드백 전환 검토 (v2) |
| situation | TensorFlow 기반 DKT 모델로 학습자 지식 상태 예측 중 (AUC 0.83). 예측 결과가 확률값(0~1)으로만 출력되어 학습자가 "왜 이 단원이 부족한지" 이해하기 어려움. 수치 기반 출력의 설명 가능성 부족 |
| option_1_name | DKT 유지 (수치 출력) |
| option_1_pros | 검증된 성능 (AUC 0.83), 가벼운 인프라 (TensorFlow Serving), 빠른 응답 속도, 예측 정확도 안정적 |
| option_1_cons | 수치만 출력 (0.72 같은 확률값), 학습자가 "왜 부족한지" 이해 불가, 피드백의 교육적 가치 낮음 |
| option_2_name | LLM 전환 (자연어 피드백) |
| option_2_pros | 자연어로 부족한 이유와 학습 방향 설명 가능, 학습자 이해도 향상, 개인화된 피드백 제공 |
| option_2_cons | API 비용 (GPT-4 기준 호출당 ~$0.03), 응답 속도 저하 (수초 단위), 환각(Hallucination) 위험, 수학 도메인 정확도 검증 필요 |
| option_3_name | 하이브리드 (DKT 예측 + LLM 피드백 생성) |
| option_3_pros | DKT의 정확한 예측값을 LLM이 자연어로 해석, 양쪽 장점 활용 |
| option_3_cons | 파이프라인 복잡도 증가, 두 시스템 관리 필요, LLM이 DKT 결과를 잘못 해석할 위험 |
| final_choice | v2에서 하이브리드 방식 파일럿 검토 예정 |
| reasoning | v1에서 DKT 수치 출력의 교육적 한계를 인지. v2에서 "DKT 예측값 + LLM 피드백 생성" 하이브리드 방식을 파일럿 테스트 계획. DKT의 예측 정확도는 유지하면서 LLM으로 설명 가능성을 보완하는 방향. 비용과 환각 위험은 파일럿에서 검증 |
| insight_1 | AI 모델 선택은 정확도 vs 설명가능성 vs 비용의 트레이드오프. 사용자(학습자)에게 전달되는 결과라면 정확도만으로는 불충분하고, 해석 가능성까지 고려해야 함 |
| insight_2 | v1의 한계를 인식하고 v2 방향을 설정하는 것도 엔지니어링. "수치 → 자연어"라는 구체적 개선 목표를 도출한 것이 기술적 판단의 결과 |
| followup_q1 | DKT 예측값을 LLM에 전달할 때 프롬프트는 어떻게 설계할 계획인가? |
| followup_q2 | LLM 환각 문제를 수학 도메인에서 어떻게 검증하려고 했나? |

## 트레이드오프 맥락 (면접 대비용)

상황에서 핵심 제약 조건:
- 제약 1: DKT 모델 성능은 AUC 0.83으로 검증 완료, 예측 자체는 문제 없음
- 제약 2: 출력이 확률값(0~1)뿐이라 학습자에게 교육적 피드백 불가
- 제약 3: LLM 전환 시 비용, 속도, 환각 위험이라는 새로운 제약 발생

각 선택지를 고려한 이유:
- DKT 유지: 검증된 성능, 변경 비용 0 → but 교육적 가치 낮음
- LLM 전환: 설명 가능성 획득 → but 비용/속도/정확도 새로운 리스크
- 하이브리드: 양쪽 장점 활용 → but 시스템 복잡도 증가

최종 선택의 결정적 근거:
- 교육 서비스에서 "왜 부족한지"를 설명하지 못하면 핵심 가치 부족
- DKT의 예측 정확도를 버리지 않으면서 LLM으로 설명 가능성 보완
- v2 파일럿으로 비용/환각 위험을 사전 검증 후 결정

만약 상황이 달랐다면:
- 사용자가 교사(전문가)였다면 → 수치만으로 충분, DKT 유지
- 실시간 응답이 필수였다면 → LLM 응답 속도 문제로 DKT 유지
- 비용 제약이 없었다면 → LLM 전면 전환 후 성능 최적화
